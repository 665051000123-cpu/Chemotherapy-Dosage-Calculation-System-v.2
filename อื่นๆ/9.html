Oncology Dose Verification System Blueprint

3. Folder Structure

โครงสร้างโฟลเดอร์โปรเจกต์

ระบบคำนวณโดสยาเคมีบำบัด
📦 oncology-dose-system/ (React + Node.js + MySQL)
📂 client/ (ส่วนหน้าบ้าน - React / Vite)
├── 📂 public/
└── 📄 logo.png
└── 📂 src/
├── 📂 components/ (หน้าจอระบบ)
├── 📄 Login.jsx
├── 📄 AdminUsers.jsx
├── 📄 ChangePassword.jsx
└── 📄 Notification.jsx
└── 📂 utils/ (ฟังก์ชันคำนวณโดสยา)
📂 oncology-backend/ (ส่วนหลังบ้าน - Node.js)
├── 📂 src/ (ลอจิกควบคุม API หลังบ้าน)
├── 📄 .env (ไฟล์เก็บความลับระบบ)
├── 📄 server.js (ไฟล์หลักเปิด API Server)
└── 📄 start-server.bat
📂 SQL/ (ฐานข้อมูล - MySQL Database)
├── 📄 user.sql
├── 📄 ข้อมูลยา.sql
├── 📄 ประวัติคนไข้.sql
├── 📄 ประวัติการแก้ไข.sql (Audit Trail)
└── 📄 ประวัติการเข้าสู่ระบบ.sql
📋 หลักการจัดโครงสร้างระบบ
💻 แยก Client / Backend ชัดเจน

แยกซอร์สโค้ดหน้าบ้านและหลังบ้านออกจากกัน เพื่อความปลอดภัยในการประมวลผลข้อมูลสูตรยาเคมีบำบัด และง่ายต่อการบำรุงรักษา

🔑 ซ่อนไฟล์ .env อย่างรัดกุม

ไฟล์ตั้งค่าความลับระบบ (เช่น Database Credentials หรือ Secret Keys) จะถูกเก็บไว้ใน .env หลังบ้าน และไม่ถูกเปิดเผยต่อสาธารณะเด็ดขาด

🛡️ เก็บ Log ประวัติ (Audit Trail)

มีตารางเฉพาะสำหรับเก็บบันทึกประวัติการแก้ไขและประวัติการเข้าสู่ระบบ เพื่อรองรับมาตรฐานความปลอดภัยทางคลินิก (Clinical Safety)

⚡ จัดหมวดหมู่โมดูลคำนวณ

แยกกลุ่มฟังก์ชันช่วยเหลือกลาง (utils) สำหรับคำนวณโดสยาเคมีบำบัดออกจาก UI เพื่อให้สามารถทำ Unit Test สอบทานความแม่นยำได้ง่าย
